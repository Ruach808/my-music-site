🔥 Firebase Integration — מדריך חיבור לייצור

הפלטפורמה בנויה עם שכבת נתונים מודולרית. להפיכתה לפעילה גלובלית, בצע את הצעדים הבאים:

שלב 1 — הוסף את פרטי Firebase בראש ה-<script> הראשי:

// ══ FIREBASE CONFIG — הדבק כאן את פרטי הפרויקט מ-console.firebase.google.com ══ const FIREBASE_CONFIG = { apiKey: "AIzaSy...", // Project Settings → General → Web API Key authDomain: "your-app.firebaseapp.com", projectId: "your-app", storageBucket: "your-app.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };

שלב 2 — החלף את שכבת localStorage בקריאות Firestore:

// במקום: function getArtists(){ return lsGet('artists',[]); } // הוסף: import { initializeApp } from "https://www.gstatic.com/firebasejs/10.x/firebase-app.js"; import { getFirestore, collection, getDocs, addDoc, setDoc, doc } from "https://www.gstatic.com/firebasejs/10.x/firebase-firestore.js"; const app = initializeApp(FIREBASE_CONFIG); const db = getFirestore(app); // DB.getArtists → קריאת כל האמנים מ-Firestore async function DB_getArtists() { const snap = await getDocs(collection(db, "artists")); return snap.docs.map(d => ({ id: d.id, ...d.data() })); } // DB.saveArtist → שמירת אמן ב-Firestore async function DB_saveArtist(artistData) { const ref = doc(db, "artists", artistData.ownerUsername); await setDoc(ref, artistData, { merge: true }); } // DB.addReport → הוספת דיווח ל-Firestore async function DB_addReport(reportData) { await addDoc(collection(db, "reports"), reportData); }

שלב 3 — Collections הנדרשים ב-Firestore:

Firestore Collections: ├── users/ {username} → user object ├── artists/ {ownerUsername} → artist profile ├── reports/ auto-id → report object ├── messages/ {threadId} → message array ├── comments/ {profileUsername} → comment array └── adminlog/ auto-id → log entry

⚠️ אבטחה: הגדר Firestore Security Rules כדי שמשתמשים יוכלו לקרוא רק נתונים ציבוריים ולכתוב רק לנתונים שלהם. לעולם אל תכלול API keys ב-client-side code ללא הגבלת דומיין ב-Firebase Console.
